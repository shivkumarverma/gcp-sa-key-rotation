Introduction
What Is This System?
Every application running on Google Cloud uses a service account key — a digital credential that proves the app's identity and grants it access to resources. Like any password, these keys must be regularly changed to stay secure. When managed manually across dozens of accounts and projects, this process is slow, error-prone, and often missed — creating security gaps and the risk of sudden application outages.

This automation handles the entire process for you — scanning, rotating, validating, and reporting — every week, without any manual effort.

Executive Summary & User Guide
Overview & How It Works
1
Scan
Reviews all service account credentials across every configured GCP project.
2
Detect
Flags credentials expiring within 14 days for proactive rotation.
3
Rotate
Generates new secure credentials and stores them in a protected vault.
4
Report
Sends a full status report via email to all stakeholders.
What You Receive After Each Run
  • HTML Email Report — colour-coded summary of all service accounts: which were rotated, which are healthy, and which need attention.
  • Excel Attachment — full detail for every credential: project, account name, rotation date, status, and storage location.
  • Delivered automatically — no action needed to receive the report. It arrives after every weekly run.
Your 3-Step Guide (After Receiving the Report)
  • Review the report — identify any service accounts with a "Rotated" status. These are the ones that need new credentials distributed.
  • Coordinate with your team — the DevOps team retrieves the new credentials from the secure vault and passes them to the relevant application team.
  • Confirm & close — the application team updates their system with the new credentials and confirms everything is working. The old key is then retired.
